# Reference Layout Notes

## Purpose

The user provided a reference screenshot from a Castle Wars-like interface to discuss layout clarity.

This screenshot is **not** a shippable asset and should not be copied. It should only inform broad interaction principles.

## What Works In The Reference

### 1. The Board Is Immediately Understandable

The player sees:

- Their hand.
- Opponent side.
- Fortress/wall state.
- Resource state.
- Draw/played-card area.
- Current or recent card.

This makes the match easy to follow without reading a log.

### 2. Fortresses Are Physical

Castle and wall are represented as actual structures in the scene, not only numbers.

For Arcane Siege:

- Use Citadel and Ward.
- Keep numeric values.
- Add original visual structures.
- Make damage/build visually affect the structures.

### 3. Card Zones Are Clear

The reference makes it visually clear that cards exist in zones:

- Hand.
- Draw stack.
- Played/current card.
- Returned/used area.

For Arcane Siege:

- Use Portmanteau draw stack.
- Use Return/recycle pulse.
- Use Last Cast focus.
- Avoid a permanent discard pile unless rules change.

### 4. Opponent Visibility Feels Natural

The opponent side can show hidden cards face down while preserving board awareness.

For Arcane Siege:

- AI hand should be face down.
- Local secret hot-seat should hide inactive hand.
- Local open-table can reveal both hands.

### 5. The Current Action Has A Place

The reference gives the played/current card a visual location, which makes turns easier to follow.

For Arcane Siege:

- Last-cast focus is mandatory in v2.
- AI turns especially need this because players are watching rather than clicking.

## What Not To Copy

Do not copy:

- Exact background composition.
- Exact UI sidebars.
- Exact icons.
- Exact castle/wall art.
- Exact card frame art.
- Exact button layout/chrome.
- Any image or screenshot.

## Arcane Siege Translation

The v2 version should feel like:

- A ritual table rather than a snowy battlefield.
- Citadels and Wards rather than castle and wall.
- Portmanteau piles rather than generic decks.
- Talisman cards rather than copied card frames.
- Law/Neutral/Chaos magical systems rather than brick/crystal/sword resource panels.

The broad lesson is spatial clarity, not visual copying.
